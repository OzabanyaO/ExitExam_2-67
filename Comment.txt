Model
Suit (abstract class) เป็นคลาสฐานที่เก็บคุณสมบัติพื้นฐานของชุด เช่น code และ durability มีเมธอด abstract เช่น isDurabilityValid() และ getSuitType() ซึ่งให้แต่ละประเภทของชุดกำหนดเงื่อนไขของตัวเอง
PowerSuit, StealthSuit, IncognitoSuit (subclasses) เป็นคลาสลูกของ Suit แต่ละคลาสจะมีการกำหนดเงื่อนไขการตรวจสอบความทนทานที่แตกต่างกัน
SuitRepository จำลองฐานข้อมูลโดยใช้ไฟล์ suits.csv เพื่อเก็บข้อมูลชุดซุปเปอร์ฮีโร่ มีหน้าที่สร้างไฟล์ CSV เมื่อไม่มีอยู่ และโหลดข้อมูลจาก CSV เพื่อแปลงเป็นวัตถุของ Suit มีเมธอดค้นหาชุดตามรหัส

View
SuitInputPanel เป็นหน้าจอที่แสดงในส่วนของการรับข้อมูลจากผู้ใช้ ให้ผู้ใช้กรอก code ผ่านช่องข้อความและกดปุ่ม "Check Suit" หน้าที่คือรับอินพุตจากผู้ใช้แล้วส่งไปยัง Controller ตรวจสอบความถูกต้องของข้อมูล
SuitResultPanel เป็นหน้าจอที่แสดงรายละเอียดของชุดที่ค้นพบ พร้อมทั้งแสดงผลการตรวจสอบความทนทาน หากชุดไม่ผ่านการตรวจสอบจะเปิดใช้งานปุ่ม "Repair Suit" ให้ผู้ใช้คลิกเพื่อซ่อมแซมชุด มีปุ่ม "Back" ให้กลับไปยังหน้า Input Panel
SuitFrame เป็นหน้าต่างหลักของโปรแกรมที่ใช้ CardLayout ในการสลับระหว่างหน้าจอ มีส่วนแสดงผลสรุปการซ่อมแซมที่แสดงจำนวนชุดที่ถูกซ่อมแซมในแต่ละประเภทอยู่ด้านล่างของหน้าต่าง

Controller
SuitController เป็นตัวกลางระหว่าง Model กับ View รับข้อมูลจาก View แล้วตรวจสอบความถูกต้องของรหัส ค้นหาชุดจากฐานข้อมูลผ่าน SuitRepository หากไม่พบหรือข้อมูลไม่ถูกต้อง จะส่งข้อความแจ้งความผิดพลาดผ่าน View หากชุดพบและไม่ผ่านการตรวจสอบความทนทาน จะเปิดให้ผู้ใช้ซ่อมแซม นอกจากนี้ยังเก็บสถิติการซ่อมแซมชุดในแต่ละประเภทไว้ใน Map ซึ่งจะถูกอัพเดทและแสดงผลในส่วนของ Repair Summary ใน View

Main เป็น entry point ของโปรแกรม ใช้ SwingUtilities.invokeLater() เพื่อสร้าง GUI ให้ทำงานบน Event Dispatch Thread สร้าง Object ของ SuitRepository และ SuitFrame จากนั้นสร้าง SuitController และเชื่อมโยงเข้าด้วยกัน
